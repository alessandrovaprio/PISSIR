#### Gruppo ingegneristico UPO-IoT## Pregettazione: Sistema di Gestione Eventi, Aule e Responsabili### Sottosistema Gestione Eventi#### IntroduzioneQuesto sottosistema si occupa della gestione degli eventi della struttura universitaria.<br>La sua interfaccia fornisce i metodi per creare, eliminare, modificare e visualizzare gli eventi della struttura universitaria. <br>Il corpo si occupa di far interagire i metodi forniti con l'interfaccia con il database degli eventi.#### ClassiIl funzionamento di questo sottosistema si basa sulle classi:- Evento - GestioneEvento- GestioneEventoDatabase##### EventoLa classe Evento rappresenta l'oggetto evento. Le informazioni che l'oggetto evento deve contenere sono:- il suo codice identificativo;- il codice identificativo del suo responsabile;- il codice identificativo dell'aula in cui ha sede;- il tipo d'evento;- il numero massimo di partecipanti;- il numero di presenti in quel momento (se l'evento è in corso);- il grado di riempimento dell'aula;- la data di svolgimento;- la fascia oraria di svolgimento;- una variabile che indica se l'evento è in corso o è terminato.##### GestioneEventoLa classe GestioneEvento permette di gestire gli oggetti di tipo evento. Inoltre per ogni operazione interagisce con la classe GestioneEventiDatabase per dialogare con il database che si occupa di memorizzare gli eventi dell'università.<br>Le operazioni consentite sono:- creazione di un evento;- eliminazione di un evento;- modifica di un evento.- caricamento di un evento dal database;- ricerca di un evento nel database.##### GestioneEventoDatabaseLa classe GestioneEventoDatabase si occupa di tradurre in query sql le operazioni richieste tramite la classe GestioneEvento. Le operazioni consentite sono:- creazione di un evento;- eliminazione di un evento;- modifica di un evento.- caricamento di un evento dal database;- ricerca di un evento nel database.
